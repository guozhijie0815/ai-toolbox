# UI 设计参考

本次 UI 优化参考 `VoltAgent/awesome-design-md` 中的三个设计系统，按页面区域拆分使用，不直接照搬完整风格。

| 区域 | 参考 | 用法 |
| --- | --- | --- |
| 主色、背景、按钮、弹窗 | Claude | 使用暖白画布、陶土橙主按钮、低对比细边框、8px/12px 圆角 |
| 搜索框、列表密度、工具卡片 | Cal.com | 使用紧凑输入框、轻量卡片、少阴影、清晰扫描节奏 |
| 表格、配置列表、右侧仓库面板 | Airtable | 使用数据管理型列表密度、浅色表头、紧凑标签、右侧面板式层级 |

## 当前项目落地规则

| 设计项 | 规则 |
| --- | --- |
| 背景 | 页面保持暖白，不使用大面积渐变和强装饰 |
| 主按钮 | 陶土橙，8px 圆角，hover 只做轻微加深和弱阴影 |
| 卡片 | 8px 圆角、1px 细边框、白色或浅暖色底，避免悬浮感过重 |
| 列表 | 保持高信息密度，卡片内边距控制在 10px-14px |
| 搜索 | 输入框高度 36px-42px，边框清晰，聚焦态使用浅橙描边 |
| 弹窗 | 内容区域用浅暖色块分组，页脚按钮固定右侧 |
| 表格 | 表头浅底、正文白底，操作按钮使用图标按钮 |
| 标签 | 小尺寸、低饱和背景，用于状态和来源，不抢主内容 |

## 不采用

| 方向 | 原因 |
| --- | --- |
| 整体暗色科技风 | 与现有浅色暖调差异过大 |
| 大字号官网式 hero | 当前是工具型应用，不需要营销页结构 |
| 高饱和多色渐变 | 会削弱管理工具的可读性 |
| 重阴影和玻璃拟态 | 与 Claude / Cal.com 的轻量产品感不一致 |
